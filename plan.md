### 阶段一：全盘系统设计方案（Tech Design）

基于你的开题报告，我将你的系统解构为以下几个核心技术落地层：

#### 1. 数据库层 (MySQL)

这是整个系统的灵魂。我们要设计规范的表结构，并加入开题报告中提到的特定字段：

* **权限模块 (RBAC):** `sys_user` (用户), `sys_role` (角色), `sys_menu` (菜单/权限), `sys_user_role`, `sys_role_menu`。
* **核心业务模块:**
* `ast_category` (资产分类：决定折旧年限和残值率)。
* `ast_asset` (资产主表：包含 **UUID**, 状态, **Version(乐观锁)** 等字段)。
* `ast_record` (资产流转/调拨记录表)。
* `ast_maintenance` (维修记录表)。
* `ast_inventory` (盘点记录表)。



#### 2. 后端核心架构 (Spring Boot)

* **鉴权中心:** Spring Security + JWT 过滤器拦截请求，解析 Token 获取用户身份和角色，实现业务/财务权限隔离。
* **持久层:** MyBatis-Plus 自动注入，配置乐观锁插件（拦截 `@Version` 字段），配置自动填充插件（`create_time`, `update_time`）。
* **核心算法层:**
* **折旧引擎:** 使用 `@EnableScheduling` 和 `@Scheduled` 定时任务，每月 1 号凌晨自动计算在用资产的折旧，写入财务表。
* **二维码引擎:** 引入 Google `ZXing` 依赖，将资产 UUID 生成二维码 Base64 字符串返回给前端。



#### 3. 前端展示层 (Vue2/3 + Element UI/Plus)

* **基础架构:** Vue Router 控制动态路由（根据 RBAC 菜单权限渲染左侧菜单），Axios 封装请求拦截器（自动在 Header 挂载 JWT）。
* **可视化大屏:** 引入 ECharts，请求后端聚合接口，展示“资产状态饼图”、“各部门资产总值柱状图”、“资产闲置趋势折线图”。
* **核心页面:** 资产台账列表（支持高级查询、分页、查看二维码图片）、入库表单、流转审批弹窗。
